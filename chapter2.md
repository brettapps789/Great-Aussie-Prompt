# Chapter 2: System Instructions for the Australian Jurisdiction

## Hard-Coding Compliance

The true power of the Sovereign Aussie AI Workforce lies in its 'System Instructions.' While a standard prompt is a request, a system instruction is a constitution. It defines the model's identity, its boundaries, and its primary source of truth. For the Aussie Architect, this means hard-coding Australian compliance directly into the AI's persona.

### The Australian Compliance Stack
1.  **Jurisdictional Primacy:** Every system instruction must explicitly state that Australian Federal and State laws take precedence over any 'general' or global guidelines.
2.  **Commercial Guardrails:** We define strict rules for financial advice, consumer protection (under ACL), and tax logic (GST/PAYG), ensuring the AI never provides non-compliant commercial outputs.
3.  **Data Sovereignty Protocols:** The model must be instructed to flag any request that would result in Australian data being processed or stored outside of sovereign-compliant nodes.
