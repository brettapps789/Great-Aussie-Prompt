# Chapter 4: Context Windows and Retrieval: Feeding the Aussie Agent

## The Data Hunger of the Sovereign Agent

An AI agent is only as good as the information it can hold in its immediate 'memory.' In 2026, with the advent of million-token context windows, the Aussie Architect has a massive advantage. We no longer need to rely solely on brittle fine-tuning; instead, we can feed our agents entire libraries of Australian legislation, commercial contracts, and industry standards in real-time.

### Strategies for High-Signal Feeding
1.  **Massive Context Injection:** We leverage Gemini's large context window to provide the agent with a 'sovereign library.' By injecting the full text of relevant Australian Acts (e.g., The Privacy Act 1988) directly into the prompt, the agent operates with a perfect, up-to-date memory of the law.
2.  **RAG (Retrieval-Augmented Generation):** For datasets that exceed the context window, we implement localized RAG. Using vector databases hosted on the Aussie Node, the agent can 'search' for the most relevant Australian case law or market data before formulating its response.
3.  **Contextual Pruning:** To maintain efficiency, the Aussie Architect uses advanced prompting techniques to instruct the agent to ignore global noise and focus only on the provided Australian signal.
